What KIMI Does

A Force Multiplier for Clinical Care Teams

KIMI extends the reach of care teams by providing guideline-concordant coaching, exercise programming, and continuous patient monitoring — all within the ACCESS Model framework.

🧠

Clinical Decision Support

Real-time pain phenotyping (nociceptive, neuropathic, nociplastic), 8-region red flag scanning, and evidence-based treatment recommendations via rule-based + ML hybrid engine.

💬

Conversational Coaching

LLM-powered clinical dialogue with motivational interviewing, exercise coaching, and crisis support. Temperature 0.3 for clinical safety, max 500 token responses.

📋

Personalized Care Plans

4-phase rehabilitation protocols — Foundation → Building → Strengthening → Maintenance — with PROMIS-based progression criteria and automated phase advancement.

🛡️

Safety Guardrails

4-tier clinical escalation with continuous red flag scanning. 100% sensitivity target for serious pathology. Non-bypassable safety layer at every interaction.

📊

Outcome Tracking

10 validated PROMs collected on defined schedules — weekly NPRS, biweekly disease-specific measures, monthly comprehensive assessments with MCID deterioration alerts.

🔗

EHR Integration

FHIR R4 bidirectional data exchange with Epic, Cerner, Athena. 8 FHIR resource types with real-time critical sync and nightly batch for routine updates.

Data Pipeline

How KIMI Processes Patient Data

From initial input through clinical action — every interaction passes through safety, intelligence, and documentation layers.

1

Patient Input Capture

Web/mobile interface collects pain reports, exercise feedback, PROM responses, and free-text messages from daily check-ins.

2

Validation & Standardization

Data validated against schema, normalized to FHIR-compatible formats. Checked for completeness before processing.

3

Real-Time Red Flag Scanning

Every input immediately processed by the 8-region Red Flag Scanner. Concerning findings trigger instant escalation — this step is non-bypassable.

4

Clinical Decision Engine

Pain phenotyping, PROM scoring, phase progression assessment, exercise selection, and care plan updates via rule-based + ML algorithms.

5

Knowledge Base Retrieval

BERT-based semantic search against 28 clinical content files for relevant guidelines, exercises, education content, and contraindications.

6

Conversation Engine Response

LLM synthesizes clinical context into empathetic, plain-language patient responses using motivational interviewing framework (OARS).

7

Documentation & EHR Sync

All interactions logged for CMS audit trail. Clinical data exported via FHIR bridge to Epic/Cerner/Athena. Population analytics computed.
